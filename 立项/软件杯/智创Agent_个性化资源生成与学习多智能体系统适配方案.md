# 智创Agent 个性化资源生成与学习多智能体系统适配方案

适配赛题：软件杯 A3 - 基于大模型的个性化资源生成与学习多智能体系统开发

---

## 一、定位

智创Agent 在软件杯中的定位：

> 面向计算机课程项目实践的个性化学习资源生成与多智能体协作系统。系统通过自然语言对话构建动态学习画像，结合课程知识库、项目分析报告和学习目标，由多个 Agent 协同生成个性化学习资源、学习路径、练习任务、代码实操案例和项目实践材料，帮助学生更高效地完成课程学习和项目能力提升。

这一定定位与挑战杯、开源创新大赛不冲突：

- 挑战杯：强调计算机学科垂类智能体应用。
- 开源创新大赛：强调软件工程项目实践课程案例包。
- 软件杯：强调个性化资源生成与学习多智能体协同。

---

## 二、赛题要求映射

| 软件杯要求 | 智创Agent 响应 |
|---|---|
| 对话式学习画像自主构建 | 学生通过自然语言描述学习基础、目标、项目经历和困难点，系统抽取画像 |
| 不少于 6 个画像维度 | 知识基础、算法能力、工程实践、AI 应用能力、学习目标、资源偏好、易错点、项目经历 |
| 多智能体协同资源生成 | 画像 Agent、课程 Agent、资源 Agent、练习 Agent、项目 Agent、评估 Agent |
| 至少 5 种资源生成 | 讲解文档、知识点思维导图、练习题、拓展阅读、代码实操案例、项目实践材料 |
| 个性化学习路径规划 | 基于画像和目标生成阶段计划、资源清单、检查点 |
| 资源精准推送 | 根据短板、目标和学习阶段推送资源卡片 |
| 智能辅导 | RAG 问答、代码解释、项目改进建议 |
| 学习效果评估 | 通过项目分析、任务反馈、练习结果更新画像和路径 |
| 防幻觉和安全 | 引用来源、AI 生成标识、边界说明、评测记录 |
| 流式或进度追踪 | 流式回答、Agent 任务状态、长任务恢复 |

## 三、核心用户场景

### S1：对话式画像构建

学生输入：

> 我是计算机大二学生，学过 Python 和数据库，算法一般，想做一个 AI 应用项目，但不知道怎么开始。

系统输出：

- 学习目标：AI 应用开发。
- 知识基础：Python、数据库入门。
- 短板：算法基础、RAG 项目经验、测试部署。
- 资源偏好：项目实操优先。
- 推荐路径：先完成 RAG 小项目，再补测试和部署。

### S2：多 Agent 生成学习资源

学生输入：

> 帮我围绕 RAG 文档问答项目生成一套学习资料。

系统输出：

1. RAG 概念讲解文档。
2. 知识点思维导图。
3. 练习题。
4. 拓展阅读。
5. 代码实操案例。
6. 项目实践任务书。

### S3：资源推送与学习路径

系统根据画像推送：

- 第 1 阶段：Python 文件处理、向量检索基础。
- 第 2 阶段：RAG Pipeline 小实验。
- 第 3 阶段：项目分析报告接入。
- 第 4 阶段：部署、测试和答辩材料。

### S4：学习效果评估

系统读取学生的项目报告、任务完成情况和反馈，输出：

- 本阶段掌握情况。
- 仍需补强的知识点。
- 下一阶段资源推荐。
- 项目改进任务。

## 四、多智能体协同设计

| Agent | 职责 | 输入 | 输出 |
|---|---|---|---|
| 画像 Agent | 从对话、项目和反馈中抽取画像 | 学生描述、项目报告、历史任务 | 动态画像、置信度、证据 |
| 课程 Agent | 检索课程知识和项目案例 | 学习目标、课程范围、知识库 | 知识点、课程资源、引用来源 |
| 资源 Agent | 生成个性化学习材料 | 画像、知识点、资源类型 | 讲解文档、阅读材料、资源卡片 |
| 练习 Agent | 生成练习题和检查点 | 知识短板、难度要求 | 选择题、简答题、代码题 |
| 项目 Agent | 生成实操案例和项目任务 | 项目目标、技术栈、能力短板 | 代码实操案例、项目任务书 |
| 评估 Agent | 评估学习效果并调整路径 | 任务反馈、练习记录、项目报告 | 学习效果摘要、路径调整建议 |

编排方式：

```text
学生自然语言输入
  -> 画像 Agent 抽取画像
  -> 课程 Agent 检索知识库
  -> 资源 Agent 生成讲解和阅读材料
  -> 练习 Agent 生成题目和检查点
  -> 项目 Agent 生成实操案例和项目任务
  -> 评估 Agent 检查资源适配度
  -> 学习路径和资源卡片保存
```

## 五、画像维度设计

软件杯要求不少于 6 个维度，智创Agent 首版定义 8 个维度：

| 维度 | 说明 | 证据来源 |
|---|---|---|
| 知识基础 | 已学课程、概念掌握情况 | 对话、课程记录、自评 |
| 算法能力 | 数据结构、复杂度、题目训练经验 | 项目、竞赛经历、自评 |
| 工程实践 | 项目结构、代码质量、测试、部署 | 项目分析报告 |
| AI 应用能力 | RAG、Agent、模型调用、评测经验 | 项目经历、学习目标 |
| 学习目标 | 短期和长期学习方向 | 对话、个人中心 |
| 资源偏好 | 文档、题目、实操、视频脚本等偏好 | 对话、反馈 |
| 易错点 | 常见错误、薄弱知识点 | 练习结果、项目报告 |
| 项目经历 | 已完成项目和角色分工 | 项目中心、Git 仓库 |

## 六、资源类型设计

首版至少展示 6 类资源，满足软件杯“至少 5 种类型”要求。

| 资源类型 | 说明 | 示例 |
|---|---|---|
| 讲解文档 | 面向当前画像的概念解释 | RAG Pipeline 入门说明 |
| 知识点思维导图 | 用层级结构展示概念关系 | RAG：文档解析、切片、检索、生成、引用 |
| 练习题 | 选择题、简答题、代码题 | SQL 索引、API 设计、RAG 检索题 |
| 拓展阅读 | 推荐阅读材料和学习顺序 | 向量检索、工程测试、部署说明 |
| 代码实操案例 | 小型可运行代码或伪代码任务 | 构建 Markdown 文档检索器 |
| 项目实践材料 | 项目任务书、验收标准、Rubric | RAG 文档问答项目任务书 |

## 七、验收标准

| 编号 | 验收项 | 标准 |
|---|---|---|
| SC-A1 | 对话画像 | 能从一段自然语言中抽取至少 6 个画像维度 |
| SC-A2 | 多 Agent | 文档中明确至少 5 个 Agent 职责，系统工作流可追踪 |
| SC-A3 | 资源生成 | 至少展示 5 种个性化资源样例 |
| SC-A4 | 路径规划 | 能根据画像和目标生成阶段学习路径 |
| SC-A5 | 资源推送 | 不同画像能获得不同资源组合 |
| SC-A6 | 智能辅导 | 问答或项目改进建议带来源或证据 |
| SC-A7 | 效果评估 | 能根据反馈或项目报告调整画像和路径 |
| SC-A8 | 用户体验 | 支持流式回答或任务进度，避免长时间空白等待 |

## 九、结论

智创Agent 已经具备软件杯要求的基础能力：学生画像、知识库、RAG、学习路径、项目分析、多 Agent 工作流和评测记录。为了更稳地满足软件杯，需要把“个性化资源生成”显性化为核心模块，并准备 5 类以上资源样例、对话画像样例和多 Agent 协作流程说明。
