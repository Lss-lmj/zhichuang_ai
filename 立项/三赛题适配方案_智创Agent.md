# 智创Agent 三赛题适配方案

版本：v0.2

适配对象：

- 挑战杯：面向一流学科建设的学科垂类大模型与创新应用开发。
- 开源创新大赛：面向计算机等专业课程教学场景的开源教学案例与智能化教学支持能力建设。
- 软件杯：基于大模型的个性化资源生成与学习多智能体系统开发。

---

## 一、总体判断

智创Agent 不需要拆成三个作品。系统可以保持一个核心产品，只在不同比赛材料中调整叙事重点和交付证据。

统一产品内核：

> 智创Agent 是基于通用大模型、计算机学科知识库、RAG 和多 Agent 工作流的计算机学科项目实践与个性化学习智能体平台。系统以学生项目资产和学习目标为入口，将课程项目、竞赛作品、个人作品和双创项目转化为可追溯的项目报告、能力证据、个性化学习资源、成长计划和教师项目学情诊断结果。

三套赛题叙事：

| 赛题 | 汇报主线 | 重点证明 |
|---|---|---|
| 挑战杯 | 计算机学科垂类智能体应用 | 学科知识库、RAG、自然语言交互、个性化学习、项目分析、教师学情诊断、可推广平台 |
| 开源创新大赛 | 软件工程项目实践课程的开源教学案例与智能化支持平台 | 软件工程课程案例包、实验任务、Rubric、教师指导材料、课堂实践闭环、开源复用 |
| 软件杯 | 个性化资源生成与学习多智能体系统 | 对话式画像、多 Agent 资源生成、5 类以上学习资源、学习路径规划、资源推送、学习效果评估 |

一句话策略：

> 挑战杯讲“学科垂类智能体平台”，开源创新大赛讲“软件工程课程项目实践开源教学案例包”，软件杯讲“面向计算机课程学习的个性化资源生成多智能体系统”。三者共用同一套画像、知识库、RAG、项目分析、资源生成、教师看板和评测能力。

## 三、挑战杯适配口径

挑战杯的核心是“学科垂类大模型与创新应用开发”。智创Agent 的挑战杯定位保持为：

> 智创Agent·计算机学科垂类大模型与双创能力赋能平台，是面向计算机学科教学与创新实践场景的垂类智能体应用。系统不以从零训练基础模型为目标，而是通过计算机学科知识库、RAG、项目分析 Agent 和教师学情诊断工作流，形成可运行、可验证、可追溯的学科智能应用。

| 挑战杯关注点 | 本项目响应 |
|---|---|
| 特定学科 | 计算机科学与技术 |
| 学科知识库 | 课程资料、项目案例、竞赛资料、Rubric、FAQ |
| RAG 与可追溯 | 回答和推荐显示来源、更新时间、引用片段 |
| 智能工作流 | 项目分析、学习计划、竞赛推荐、组队推荐、教师诊断 |
| 多轮自然语言交互 | 学生问答、学习规划、教师诊断解释 |
| 个性化与自适应 | 基于学生画像、项目证据、目标方向和反馈调整建议 |
| 真实教学场景 | 课程项目、学生项目报告、班级项目学情看板 |
| 可复制推广 | Web 公网访问环境、学校账号入口、API、部署说明、测试记录 |

## 四、开源创新大赛适配口径

开源创新大赛更关注“课程教学场景、开源教学案例、智能化教学支持”。智创Agent 的开源创新大赛定位建议升级为：

> 智创Agent 是面向计算机专业核心课程的交互式开源教学案例平台。系统围绕深度学习框架、计算机系统、编译原理、数据库、操作系统、计算思维、软件工程和 AI 基础设施八个方向，设计为八门独立课程，每门课程包含章节和课节，提供可课堂投屏使用的 HTML 交互演示、实验任务、系统支持能力、课堂产出和证据化评价口径。

首版最适合形成完整闭环的方向仍是软件工程方向，因为它能直接复用项目管理、项目分析、教师看板和学生改进任务。但开源创新大赛材料不能只停留在软件工程单方向，需要用“八门课程开源课堂套件”覆盖赛题完整边界。当前前端与案例包按 8 门课程、32 章、96 节组织。

| 开源创新大赛要求 | 本项目响应 |
|---|---|
| 专业课程教学场景 | 覆盖八个方向的计算机专业核心课程，软件工程方向作为 P0 详案 |
| 课程体系与实践体系 | 8 门课程、32 章、96 节，每节包含课堂演示、实验任务、课堂产出和评价口径 |
| 高质量开源教学案例 | 八门课程交互式教学案例套件 + 软件工程项目实践详案 |
| 智能化教学支持 | 过程讲解 Agent、实验指导 Agent、项目分析 Agent、知识库问答、教师学情诊断 |
| 课堂互动与参与度 | 教师投屏播放过程动画，学生完成实验任务并提交项目或实验产出 |
| 教学成效验证 | 高仿真课堂流程、示例班级数据、报告生成效率和问题发现率 |
| 可复用性 | HTML 交互演示、任务书、Rubric 和课堂脚本可迁移到不同学校课程 |

### 4.1 八门课程覆盖矩阵

| 方向 | 首版课程 | 章节规模 | 交互演示重点 |
|---|---|---|---|
| 深度学习框架与编程 | 深度学习框架实践 | 4 章 12 节 | 张量、计算图、反向传播、优化器 |
| 计算机系统导论 | 计算机系统基础 | 4 章 12 节 | 指令执行、寄存器、栈帧、内存变化 |
| 编译原理 | 编译原理课程实验 | 4 章 12 节 | Token、AST、符号表、中间代码 |
| 数据库系统概论 | 数据库系统课程设计 | 4 章 12 节 | SQL 解析、优化器、索引、事务 |
| 操作系统内核构建 | 操作系统课程实验 | 4 章 12 节 | 进程调度、系统调用、页表映射 |
| 计算思维相关方向 | 计算思维与程序设计 | 4 章 12 节 | 分解、循环、分支、状态迁移 |
| 软件工程 | 软件工程项目实践 | 4 章 12 节 | 需求、设计、代码、测试、交付、复盘 |
| AI 基础设施 | AI 系统与基础设施实践 | 4 章 12 节 | 算子、编译优化、分布式通信、吞吐率 |

开源课堂前端采用分层页面设计：课程目录页用于选择方向，课程详情页展示章节，章节详情页展示课节，课节页承载具体交互工具。首版 96 个课节均具备自己的课节编号、关键问题、过程演示、实操任务、课堂互动、作品证据和评价标准；课节内支持讲授、实操、讲评三种模式切换。

| 页面层级 | 主要内容 | 课堂价值 |
|---|---|---|
| 课程目录页 | 八门课程入口、章节与课节数量 | 帮助教师快速选择本节课所在课程 |
| 课程详情页 | 课程章节列表、课程材料栈、章节入口 | 让课程结构清楚可导航 |
| 章节详情页 | 本章课节列表、课节重点、课时信息 | 让教师按教学进度进入对应课节 |
| 课节交互页 | 过程动画、实操任务、作品证据、评价标准 | 支撑现场讲授、动手练习和作品讲评 |

## 五、软件杯适配口径

软件杯的核心是“个性化资源生成与学习多智能体系统”。智创Agent 的软件杯定位建议为：

> 智创Agent 是面向计算机课程项目实践的个性化学习资源生成与多智能体协作系统。系统通过自然语言对话构建动态学习画像，结合课程知识库、项目分析报告和学习目标，由画像 Agent、资源 Agent、练习 Agent、项目 Agent、评估 Agent 协同生成讲解文档、知识图谱、练习题、拓展阅读、代码实操案例和项目实践材料，并持续调整学习路径和资源推送。

### 5.1 软件杯核心要求映射

| 软件杯要求 | 本项目响应 |
|---|---|
| 对话式学习画像自主构建 | 学生通过自然语言描述专业、目标、基础、项目经历和学习历史，系统抽取画像维度 |
| 不少于 6 个画像维度 | 知识基础、工程实践、算法能力、AI 应用能力、学习目标、资源偏好、易错点、项目经历 |
| 多智能体协同资源生成 | 画像 Agent、课程 Agent、资源 Agent、练习 Agent、项目 Agent、评估 Agent 协作 |
| 至少 5 种个性化资源 | 讲解文档、知识点思维导图、练习题、拓展阅读、代码实操案例、项目实践材料 |
| 个性化学习路径规划 | 基于画像、目标、项目证据和反馈生成阶段计划 |
| 资源精准推送 | 按知识短板、项目目标、资源偏好和学习阶段推送资源 |
| 智能辅导 | RAG 问答、代码问题解释、概念拆解、项目改进建议 |
| 学习效果评估 | 项目报告、练习记录、任务反馈和评测记录共同更新画像 |
| 防幻觉和内容安全 | 引用来源、边界说明、AI 生成标识、评测记录 |
| 流式或进度追踪 | SSE 流式回答、Agent 任务状态、长任务查询与恢复 |

### 5.2 软件杯需要补强的产品表达

现有系统已经具备画像、RAG、项目分析、学习计划和 LangGraph 基础，但软件杯要求把“资源生成”作为显性主线。因此需要补强：

1. 资源生成中心：把学习计划中的资源进一步结构化为多类型学习材料。
2. 多 Agent 资源生成流程：明确不同 Agent 的职责与输出。
3. 资源类型样例：至少准备 5 类个性化学习资源样例。
4. 画像对话构建：弱化表单感，强调自然语言抽取画像。
5. 学习效果评估：用练习结果、项目报告和任务反馈更新画像和资源推荐。

## 八、软件杯新增 P0 需求

### SC-F1：对话式学习画像构建

| 项目 | 内容 |
|---|---|
| 需求描述 | 学生通过自然语言描述学习目标、课程基础、项目经历和困难点，系统自动抽取画像字段 |
| 用户角色 | 学生 |
| 输出 | 不少于 6 个维度的动态学习画像、置信度、证据来源 |
| 验收标准 | 画像包含知识基础、工程实践、算法能力、AI 应用能力、学习目标、资源偏好、易错点、项目经历等维度中的至少 6 项 |

### SC-F2：多 Agent 个性化资源生成

| 项目 | 内容 |
|---|---|
| 需求描述 | 系统基于画像、课程内容、知识短板和学习需求，由多个 Agent 协作生成学习资源 |
| 用户角色 | 学生 |
| 输出 | 讲解文档、知识点思维导图、练习题、拓展阅读、代码实操案例、项目实践材料 |
| 验收标准 | 至少能展示 5 种资源类型，每种资源能说明生成依据和适用学生画像 |

### SC-F3：个性化学习路径与资源推送

| 项目 | 内容 |
|---|---|
| 需求描述 | 系统整合生成资源，形成阶段学习路径和资源推送清单 |
| 用户角色 | 学生 |
| 输出 | 阶段目标、学习步骤、资源卡片、项目任务、检查点 |
| 验收标准 | 不同基础和目标的学生获得差异化资源组合 |

### SC-F4：学习效果评估与动态调整

| 项目 | 内容 |
|---|---|
| 需求描述 | 系统根据任务完成、练习表现、项目分析报告和学生反馈调整画像与学习路径 |
| 用户角色 | 学生 |
| 输出 | 学习效果摘要、画像变化、下一阶段资源推荐 |
| 验收标准 | 学生反馈“基础薄弱、时间不足、已完成项目”等情况后，系统能更新路径和资源建议 |

## 十一、最终建议

系统主线保持不变：

> 学生项目中心 -> 项目分析报告 -> 能力证据 -> 个性化资源 -> 成长行动 -> 教师项目学情诊断。

为满足挑战杯，需要强调：

1. 计算机学科垂类知识库。
2. RAG 和内容可追溯。
3. 多 Agent 长流程项目分析。
4. 学生个性化学习与竞赛创新能力培养。
5. 教师项目学情诊断。

为满足开源创新大赛，需要补强：

1. 八门课程交互式开源教学案例套件。
2. 可课堂投屏使用的 HTML 动画和过程可视化演示。
3. 每门课程的章节、课节、实验任务、系统支持能力、课堂产出和评价口径。
4. 软件工程项目实践课程案例包作为 P0 详案。
5. 开源材料目录、协议说明、高仿真课堂运行记录和教师讲评材料。

为满足软件杯，需要补强：

1. 对话式动态学习画像。
2. 多 Agent 资源生成流程。
3. 至少 5 类个性化学习资源样例。
4. 个性化学习路径和资源推送。
5. 学习效果评估和动态调整记录。

结论：

> 智创Agent 可以同时适配三个赛题。挑战杯看它是“计算机学科垂类智能体应用”，开源创新大赛看它是“计算机专业八门课程交互式开源教学案例与智能体辅助授课平台”，软件杯看它是“基于大模型的个性化资源生成与学习多智能体系统”。三者共用同一套系统能力，差异主要在提交材料与叙事重点。
