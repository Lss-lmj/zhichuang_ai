面向计算机专业、电子技术专业、人工智能专业等专业课程教学场景，鼓励参赛队伍结合学科特点，围绕大学生创新型、系统型能力培养等核心需求，运用人工智能技术开展课程体系与教学实践的创新设计，重点考察参赛者在复杂知识体系下，构建高质量开源教学案例及智能化教学支持能力。推动我国高校课程教学场景与模式创新，提升实践教学质量，促进开源教育生态共建共享。
参赛队伍可围绕以下计算机核心课程方向（包括但不限于）选题建设，具体实践内容可结合课程特点自主设计：
（一）深度学习框架与编程方向：运用开源深度学习框架，如计图Jittor，设计教学案例、课程实验或大作业，融入AI工具与能力，辅助学生开展框架学习与编程实践。
（二）计算机系统导论方向：围绕如程序的机器级表示、二进制程序分析、缓冲区溢出攻击、x86模拟器设计等内容，开展项目实践和教学案例创建，并配套设计实验指导类智能体。
（三）编译原理方向：围绕如语法分析器构建、符号表管理、中间代码生成、代码优化等内容，开展项目实践和教学案例创建，结合AI工具与能力辅助编译原理难点讲解与实践操作。
（四）数据库系统概论方向：立足课程实践教学，以真实场景为驱动，可任选SQL能力培养、数据库应用开发、数据库运维等方向开展实践，设计兼具真实性、全面性和创新性的教学案例，可配套智能判分、操作指导类智能体。
（五）操作系统内核构建方向：基于开源操作系统，如NPUcore，围绕内存管理、进程管理、文件系统、I/O设备管理等内容，开展项目实践和教学案例创建，融入智能体辅助内核构建实践指导。
（六）计算思维相关方向：开发计算思维相关的创新教学案例，设计交互式智能体辅助学生体验机器程序执行过程。
（七）软件工程方向：围绕软件需求工程、软件设计、代码编写、软件测试、项目管理等内容，开展项目实践和教学案例创建，结合AI工具与能力辅助软件工程实践全流程。
（八）AI基础设施方向：基于开源基础设施软件栈，如众智FlagOS，围绕高性能算子开发、AI编译优化、分布式通信等关键技术，设计教学案例、课程实验或大作业，配套AI工具与能力辅助AI基础设施相关技术的教学与实践。

参赛作品应作品需契合专业课程特点，体现实践操作性、开源性与创新性，并突出以下能力：

利用AI技术提升课程资源开发与组织能力
面向能力培养的课程设计与实践体系构建
创新教学模式，增强课堂互动性与学习参与度
体现技术与教育融合的实践价值与创新思路


# 评分标准
| 评分维度 | 权重 | 核心考核点 |
| ---- | ---- | ---- |
| 教学创新与融合 | 25% | 有机融入：评估AI技术与教学场景的自然结合度，避免技术堆砌；<br>模式升级：基于传统教学模式进行智能化迭代与创新，实现从“经验驱动”向“数据智能驱动”的教学范式跃迁。 |
| 应用成效 | 25% | 数据实证：有量化数据证明学生在参与度、知识掌握度或整体教学效率上的显著提升；<br>落地反馈：已在真实课堂或高仿真场景中运行，并获得师生积极评价。 |
| 课程质量 | 25% | 逻辑自洽：教案、资源与智能体设计严密，形成完整闭环；<br>资源优质：教学内容深入浅出，资源呈现形式生动，易学易懂，显著降低师生认知负荷。 |
| 可复用性 | 25% | 通用迁移：案例解耦特定私有数据，具备跨课程、跨学科的快速迁移能力；<br>推广潜力：架构规范，具备成为行业标准化开源教学案例的潜力。 |
