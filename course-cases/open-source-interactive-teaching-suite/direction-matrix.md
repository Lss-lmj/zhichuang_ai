# 八门课程与八方向赛题映射矩阵

## 总体设计原则

1. 每个方向必须设计为一门真实课程或课程实验。
2. 每门课程至少包含 4 个章节，每章至少 3 个课节。
3. 每个课节必须包含可课堂展示的交互动画或过程可视化。
4. 每个课节必须有学生可完成的实验任务和提交证据。
5. 每门课程必须有系统支持能力，而不是单纯播放课件。
6. 每门课程必须能形成可评价的课堂产出。

## 八门课程矩阵

| 赛题方向 | 课程场景 | 交互动画或演示界面 | 实验任务 | 系统支持能力 | 课堂产出 |
| --- | --- | --- | --- | --- | --- |
| 深度学习框架与编程 | 深度学习框架实践 | 张量流动、计算图、loss、backward、optimizer 分步动画 | 补全二分类训练脚本，解释张量 shape，比较学习率变化 | 框架概念解释、代码逐行讲解、训练异常定位 | 训练代码、loss 曲线、实验报告 |
| 计算机系统导论 | 计算机系统基础 | 指令取指、译码、寄存器、栈帧、内存、返回地址变化 | 分析函数调用栈和边界写入风险，完成迷你模拟器日志 | 汇编解释、寄存器状态追踪、风险代码定位 | 指令日志、栈帧图、风险修复说明 |
| 编译原理 | 编译原理课程实验 | 源代码到 token、AST、符号表、IR、优化过程动画 | 实现表达式解析器，维护符号表，生成三地址码 | 语法错误解释、AST 结构生成、符号表检查 | 解析器代码、AST 输出、IR 结果 |
| 数据库系统概论 | 数据库系统课程设计 | SQL 解析、优化器、索引选择、执行器、事务提交路径 | 设计项目管理场景数据表，编写查询，解释执行计划 | SQL 讲解、执行计划分析、索引建议、智能判分 | ER 图、建表 SQL、查询脚本、执行计划说明 |
| 操作系统内核构建 | 操作系统课程实验 | 就绪队列、CPU 调度、系统调用、页表映射、设备 I/O 动画 | 补全系统调用，记录进程轮转，解释虚拟地址映射 | 内核代码导读、实验卡点定位、调用路径解释 | 内核补丁、调用路径图、调试日志 |
| 计算思维相关方向 | 计算思维与程序设计 | 输入、分解、循环、分支、状态迁移、输出过程动画 | 用状态表描述排序或搜索过程，根据错题生成同类练习 | 算法过程讲解、错因提示、练习生成 | 状态迁移表、算法轨迹、练习记录 |
| 软件工程 | 软件工程项目实践 | 需求、设计、代码、测试、交付、复盘项目闭环界面 | 上传项目，绑定作者和团队分工，保存改进事项 | 项目分析、需求检查、测试建议、团队分工分析 | 项目仓库、项目分析报告、班级讲评材料 |
| AI 基础设施 | AI 系统与基础设施实践 | 算子、kernel、计算图、编译优化、AllReduce、吞吐率动画 | 比较算子性能，观察通信过程，解释吞吐率瓶颈 | 算子讲解、性能瓶颈定位、通信过程解释 | 算子代码、性能记录、通信示意、优化建议 |

## 首版实现重点

首版不需要一次性把八个方向都做成完整后端评测系统，但前端课堂演示、章节目录、课节内容、实验任务、系统支持能力和评价口径必须完整。

优先级建议：

| 优先级 | 方向 | 原因 |
| --- | --- | --- |
| P0 | 软件工程 | 已与项目管理、项目分析、教师看板主线最贴合 |
| P0 | 数据库系统概论 | 与项目案例、SQL、后端数据模型强相关，容易形成可运行实验 |
| P0 | 计算机系统导论 | 动画展示价值高，适合作为课堂演示亮点 |
| P1 | 编译原理 | AST 和 IR 可视化适合做交互动画 |
| P1 | 操作系统内核构建 | 可做系统调用和页表映射动画 |
| P1 | 深度学习框架与编程 | 可做训练过程和计算图动画 |
| P1 | 计算思维相关方向 | 适合低年级课程推广 |
| P1 | AI 基础设施 | 适合作为技术先进性亮点 |

## HTML 动画与交互演示规划

| 动画主题 | 适配方向 | 课堂价值 |
| --- | --- | --- |
| 自动求导计算图 | 深度学习框架与编程 | 把训练代码和梯度传播过程对应起来 |
| 指令执行与栈帧变化 | 计算机系统导论 | 直观看到寄存器、栈、内存如何变化 |
| 语法分析与 AST 构建 | 编译原理 | 把文法规则、树结构和中间表示串起来 |
| SQL 执行计划 | 数据库系统概论 | 帮助学生理解优化器、索引和执行代价 |
| 进程调度与页表映射 | 操作系统内核构建 | 降低内核机制学习门槛 |
| 算法状态机 | 计算思维相关方向 | 帮助初学者理解循环、分支和状态迁移 |
| 项目工程闭环 | 软件工程 | 把需求、设计、测试、复盘落到项目证据 |
| 算子与分布式通信 | AI 基础设施 | 解释性能瓶颈、通信同步和吞吐率 |

## 验收口径

1. 教师能在系统中选择八门课程之一进入课堂演示。
2. 每门课程至少包含 4 个章节、12 个课节。
3. 每个课节展示教学重点、过程可视化、课堂互动、实验任务和提交证据。
4. 每门课程都列出系统支持能力和课堂产出。
5. 软件工程方向能与项目上传、项目分析报告和教师看板连通。
