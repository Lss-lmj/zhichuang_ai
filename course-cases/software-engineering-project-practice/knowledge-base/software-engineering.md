# 软件工程项目实践知识库样例

## 需求分析

需求分析关注用户是谁、要解决什么问题、核心流程是什么、哪些功能进入首版。课程项目中建议使用“用户场景 + 功能清单 + 优先级”的轻量形式。

## 软件设计

软件设计需要说明模块边界、数据模型、接口关系和关键交互流程。对于 Web 项目，建议至少说明前端页面、后端接口、数据库实体和权限边界。

## 项目管理

项目管理关注团队分工、里程碑、任务状态和风险记录。课程项目不要求复杂管理工具，但需要能说明每位成员的职责和阶段产出。

## 文档规范

README 至少应包含项目简介、技术栈、启动方式、依赖安装、核心功能、目录说明和测试方式。
