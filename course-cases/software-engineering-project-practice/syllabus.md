# 软件工程项目实践课程大纲

## 课程定位

本课程面向计算机类学生，通过一个完整项目实践，训练学生从需求分析、软件设计、代码实现、测试交付到项目讲评的全流程能力。

## 教学目标

学生完成课程后应能够：

1. 从真实或高仿真场景中抽取软件需求。
2. 设计合理的软件模块、数据模型和接口。
3. 使用团队协作方式完成可运行项目。
4. 编写运行说明、测试样例和项目文档。
5. 根据项目分析报告定位问题并完成迭代改进。
6. 理解 AI 工具在软件工程实践中的辅助价值和边界。

## 教学单元

| 单元 | 主题 | 产出 |
|---|---|---|
| U1 | 项目选题与需求分析 | 用户场景、功能清单、优先级 |
| U2 | 原型与交互流程 | 页面流程、核心操作路径 |
| U3 | 架构与模块设计 | 模块划分、技术栈、目录规划 |
| U4 | 数据库与接口设计 | 数据模型、接口清单、权限边界 |
| U5 | 代码实现与协作 | 代码仓库、核心功能、团队分工 |
| U6 | 测试与工程规范 | README、依赖说明、测试样例 |
| U7 | 项目分析与改进 | 项目分析报告、改进事项 |
| U8 | 课堂讲评与总结 | 班级共性问题、项目展示、课程总结 |

## 建议周期

| 周次 | 教学重点 |
|---|---|
| 第 1 周 | 选题、需求分析、团队分工 |
| 第 2 周 | 原型、模块设计、数据模型 |
| 第 3-5 周 | 代码实现、测试、文档补齐 |
| 第 6 周 | 项目提交、智能分析、问题修复 |
| 第 7 周 | 教师讲评、学生展示、改进提交 |
| 第 8 周 | 课程总结、项目报告归档 |
