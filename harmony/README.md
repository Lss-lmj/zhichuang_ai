# HarmonyOS 客户端说明

鸿蒙端作为 P1 客户端单独建设，使用 ArkTS + ArkUI。它不从 React Web 页面直接迁移，而是复用后端 API、认证协议、任务状态和报告数据结构。

## 1. 定位

鸿蒙端优先服务学生端高频轻量场景：

- 查看个人能力画像。
- 查看学习路径。
- 查看作业分析报告。
- 接收任务提醒。
- 查看竞赛建议。
- 发起简单问答。

教师看板、知识库管理、作业批量分析仍以 Web 端为主。

## 2. 接口复用

鸿蒙端复用：

- `/api/agent/chat`
- `/api/students/me/profile`
- `/api/assignments/{assignment_id}/reports/{student_id}`
- `/api/plans/*`
- `/api/competitions/recommend`
- `/api/tasks/{task_id}`

## 3. 页面规划

| 页面 | 说明 |
| --- | --- |
| 登录页 | 演示账号和正式账号入口 |
| 首页 | 今日任务、最近报告、学习路径摘要 |
| 画像页 | 能力维度、证据、建议 |
| 报告页 | 作业分析摘要、评分、问题和建议 |
| 计划页 | 阶段计划、任务状态、复盘入口 |
| 问答页 | 学科智能体对话 |

## 4. 开发约定

1. 后端先保证 API schema 稳定。
2. 移动端只展示必要信息，复杂图表留在 Web。
3. 长任务通过 `task_id` 查询状态。
4. 报告和计划支持离线缓存作为后续优化项。
