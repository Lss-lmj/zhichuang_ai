# 智创Agent 三赛题验收矩阵

更新时间：2026-07-07

本文档用于说明智创Agent 如何同时满足挑战杯、开源创新大赛和软件杯三个方向的要求。

---

## 一、总体结论

智创Agent 采用“一个系统，三套赛题叙事”的交付策略。

| 赛题 | 适配结论 | 核心证据 |
|---|---|---|
| 挑战杯 | 符合，主线为计算机学科垂类智能体应用 | 计算机学科知识库、RAG、项目分析 Agent、学生成长闭环、教师项目学情诊断 |
| 开源创新大赛 | 符合，需要突出软件工程课程案例包 | 软件工程项目实践课程案例包、Rubric、任务书、教师指南、学生指南、项目分析报告样例 |
| 软件杯 | 基本符合，需要突出资源生成与多智能体协同 | 对话式画像、个性化学习路径、资源生成案例包、5 类以上资源样例、多 Agent 工作流 |

统一主线：

> 学生项目中心 -> 项目分析报告 -> 能力证据 -> 个性化资源 -> 成长行动 -> 教师项目学情诊断。

## 五、评分维度对照

### 5.1 挑战杯评分关注

| 评分维度 | 当前支撑 | 还需补强 |
|---|---|---|
| 作品完成度 | Web 前端、FastAPI 后端、SQLite 持久化、测试与构建通过 | 公网环境最终部署截图和访问说明 |
| 创意实用度 | 项目分析驱动学生成长与教师学情诊断 | 真实教师/学生试用记录 |
| 技术实现度 | RAG、LangGraph、项目分析、权限、评测 | 更完整的代码运行沙箱可作为后续增强 |
| 技术先进性 | 多 Agent 项目分析、证据画像、引用校验 | 模型评测和多模型对比结果 |
| 内容质量度 | 典型测试案例、引用来源、报告证据片段 | 标准答案对比和人工评分记录 |
| 商业化潜力 | 学校账号、课程班级、教师看板、可部署 API | 学校部署路线和运行说明 |
| 用户认可度 | 可通过高仿真课堂记录支撑 | 至少 2 名目标用户结构化反馈 |

### 5.2 开源创新大赛评分关注

| 评分维度 | 当前支撑 | 还需补强 |
|---|---|---|
| 教学创新与融合 | AI 项目分析嵌入软件工程课程实践 | 课堂讲评流程截图和教师使用记录 |
| 应用成效 | 评测记录、测试用例、报告生成流程 | 量化课堂运行数据和师生反馈 |
| 课程质量 | 课程大纲、任务书、Rubric、指南 | 3 个真实或高仿真项目报告样例 |
| 可复用性 | 案例包目录、知识库样例、通用 Rubric | 迁移指南和更多课程案例 |

| 评分维度 | 当前支撑 | 还需补强 |

| 功能实现及技术要求 | RAG、LangGraph、多 Agent、项目分析、任务状态 | 前端资源生成中心页面和更多资源样例 |
