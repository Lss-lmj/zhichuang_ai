# 开源课堂独立课件建设规范

## 目标

开源课堂的每一个课节都应是可独立打开、可课堂投屏、可键盘控制的专题课件页面。课节页面不是课程页里的内容卡片，也不允许由生成器批量产出同质化页面。

## 页面标准

每个课节页面需要单独设计：

- 独立 HTML 文件和独立页面标题。
- 与本节知识点匹配的专门交互结构。
- 分步讲解、上一步、下一步、播放、重置和键盘控制。
- 至少一个可视化舞台，使用 SVG、Canvas 或 DOM 动画呈现过程。
- 课堂可用的讲解文本、任务引导和结果解释。
- 可离线运行，不依赖外部 CDN 或网络资源。

## 学科设计原则

不同学科不能使用同一种界面骨架。每门课程需要先确定自己的交互语言，再逐节设计页面。

| 课程方向 | 交互语言 | 典型界面 |
| --- | --- | --- |
| 编译原理 | 编译流水线、文法推导、树结构、符号表、IR 和优化前后对照 | token 流、NFA/DFA、LL/LR 表、AST、三地址码、基本块 DAG |
| 计算机系统导论 | 机器状态、寄存器、栈帧、机器码字段、控制流图、数据通路 | C/汇编/机器码三级联动、反汇编单步、栈帧覆盖、CPU 流水线 |
| 数据库系统 | 查询计划、关系代数、索引路径、事务时间线、锁等待图 | SQL 到执行计划、B+Tree 查找、Join 代价比较、隔离级别并发轨迹 |
| 操作系统 | 内核路径、进程状态、页表映射、文件对象、设备队列 | Trap 路径、调度队列、虚拟地址翻译、文件描述符表、I/O 中断链路 |
| 深度学习框架 | 张量形状、计算图、梯度流、训练循环、指标曲线 | shape 追踪、算子图、backward 链、优化器路径、loss 曲线 |
| 软件工程 | 项目证据、需求流、模块边界、测试矩阵、协作图 | 用户故事地图、架构依赖图、仓库结构剖析、测试覆盖矩阵、贡献图 |
| 计算思维 | 问题分解、状态转移、递归栈、搜索树、调试轨迹 | 状态表、循环不变式、BFS/DFS 轨迹、递归调用栈、错因分类 |
| AI 基础设施 | 算子执行、访存模式、编译图优化、分布式通信、性能剖析 | 算子热力图、kernel 时间线、AllReduce 环路、Profile 火焰/时间图 |

## 禁止事项

- 不使用生成器批量生成 96 个页面。
- 不用同一套卡片模板替换标题来冒充独立课件。
- 不在界面中出现需求说明稿、提示词或内部设计口吻。
- 不给尚未完成独立设计的课节标记为已完成。
